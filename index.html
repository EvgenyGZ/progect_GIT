Урок 1.

Установка и запуск. На этой стадии мы подготавливаем место для работы и выбираем проект с которым начнем работать.

  1. Выбираем директорию, где будем работать с проектом.
  2. Запускаем GITBash.
  3. Используя команду "cd" c названием пути переходим в выбранную папку.
  4. Для начала клонируем себе содержимое основной ветки master, так как на ней я буду писать инструкции1.
  5. "Git clone https://github.com/EvgenyGZ/progect_GIT.git"
  6. Именно этой командой с этим адресом мы клонируем в выбранное место именно этот проект2.
  7. После того, как проект будет успешно скачан, можно переключаться непосредственно к работе с файлами в каталоге.
  8. Мы закончили править код в файлах и теперь можно проверить, что GIT зафиксировал какие были изменения.
  9. Командой git status мы увидим список файлов в которых были изменения. Они будут выделены красным цветом.
  10. Следующий шаг, git add .4 так мы даем понять, что все изменения нужно зафиксировать.
  11. Убедиться в успешности предыдущего шага можно с помощью повторения команды git status
  12. Если всё успешно, тогда список файлов станет зеленым.
  13. Теперь мы добавляем комментарий с помощью команды git commit -m5 "Текст комментария"6.
  14. Чтобы проверить успешность данной операции выполняем комманду git status. Если всё верно - мы ничего не увидим.
  15. В конце работы нужно отправить все изменения в сетевое хранилище. Для этого мы пишем git push origin master7.
  16. Наблюдаем как файлы улетают на сервер.
  17. На этом цикл работы считается законченным.

1 - Поскольку работа подразумевается с уже созданным репозиторием, то процесс работы мы смотрим с позиции пользователя.
2 - Адрес будет меняться в зависимости от проекта и автора.
3 - Когда мы сами создаем свой репозиторий на локальной машине, пройдя 1-2-3 шаги мы выполняем команду git init, тем самым создавая рабочай каталог. и соответственно шаги 4-5-6 здесь не уместны.
4 - Обязательно добавляем . (точку), давая понять, что надо отправить все изменения, что мы сделали.
5 - -m обозначает, что комментарий мы пишем прямо из коммандной строки. Это намного проще, чем запускать текстовый редактор.
6 - Текст обязательно стоит указать детально. Чтобы позже иметь представления о том какие именно изменения были там сделаны.
7 - имена веток с которыми будет работать каждый отдельные человек мы будем проходить позже

Используемые в работе команды
init3 clone cd status add commit push
